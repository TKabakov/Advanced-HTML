iFrame embeds an HTML page into another page.

allow

    It specifies what permissions of features are available to the frame:
  • allow="fullscreen" the fullscreen mode can be activated
  • allow="geolocation" lets you access the user's location

A semicolon-separator should be used to specify more than one feature:

name

height

The default value is 150 pixels.

width

The default value is 300 pixels.

referrerpolicy

    A referrer is the HTTP header that lets the page know who is loading it. The common values are:
  • no-referrer The referrer header will not be sent.
  • origin The refferer will be limited to the origin of the referring page.
  • strict-origin The origin of the document is sent as referrer only when using the same protocol security level(HTTPS to HTTPS)

sandbox

A sandbox applies extra restrictions to the content in the inframe.

To lift particular restriction, an attribute valueis used:

  • allow-downloads Allows the user to download an item
  • allow-forms Allows the user to submit forms.
  • allow-modals The resource can open modal windows.
  • allow-orientation-lock Lets the resource lock the screen orientation.
  • allow-popups Allows popups to open.
  • allow-presentation Allows a presentation session to start.
  • allow-scripts Lets the resource run script without creating popup windows.

When the value of sandbox is empty, all restriction are applied.

src

The URL of the page to embed in the iframe

srcdoc

Let's you specify the inline HTML to be embed in the iframe

loading

This attribute let's specify if the iframe should be loaded immediately when the page loads (eager) or when iframe is displayed in the browser (lazy).

title

This attribute let's you add a description to the iframe for accessibility purposes.
